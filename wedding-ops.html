01

One Vocabulary

Select options defined once, used identically across databases. No "Photo" in one place and "Photography" in another.

02

Guest List is the Spine

Every person-related database relates back to it. No orphan records.

03

Budget + Vendors Joined

Every dollar traces to a vendor or category. Nothing floats unattributed.

04

Phase-Aware

Databases activate in stages. Irrelevant ones stay collapsed, not deleted.

05

Dashboard-First

Hub page shows live rollup metrics. Never open a database just to get a headline number.

06

Templateable

Row templates for common entries. The whole system is duplicable for reuse.

if(empty(prop("Deposit Due")), 0, dateBetween(prop("Deposit Due"), now(), "days"))

Section 07

Lifecycle Guide

The system activates in phases. Each phase has different active databases and priorities.

Phase 1 — 12+ Months

Foundation

Build the guest list, research and book key vendors, set the budget ceiling. Seed milestone tasks.

Guest List Vendors Budget Tasks
Phase 2 — 9–12 Months

Booking

Lock in vendors, pay deposits, finalize budget estimates. Send save the dates.

Vendors Budget Tasks Guest List
Phase 3 — 6–9 Months

Planning

Finalize guest list, start music curation, order invitations, register for gifts.

Guest List Music Tasks Vendors Budget
Phase 4 — 3–6 Months

Invitations & RSVPs

Mail invitations, deploy RSVP backend, match responses to guest list records, send reminders.

Guest List RSVP Responses Tasks
Phase 5 — 1–3 Months

Final Prep

Build seating chart, review guest song requests, share DJ list, send final headcount to caterer.

Seating Chart Music Tasks Guest List
Phase 6 — Week Of & Day Of

Execution

Share Day-of Timeline with vendors, Shot List with photographer, Dietary by Table with caterer.

Day-of Timeline Shot List Tasks
Phase 7 — Post-Wedding

Wrap-Up

Track gifts received, send thank-you notes, rate vendors, close out the budget.

Guest List Vendors Budget

Section 09

Build Order

Create the system in this sequence to avoid broken relations. Check off each step as you go — progress is saved to your browser.

0 of 20 steps complete
  1. Create Guest List database (no relations yet)
  2. Create Vendors database (no relations yet)
  3. Create Budget database + add Vendor relation
    Depends on: Step 2
  4. Add Budget Lines (reverse relation) to Vendors
    Depends on: Step 3
  5. Create Seating Chart database
  6. Add Table relation to Guest List → Seating
    Depends on: Steps 1, 5
  7. Add Guests (reverse relation) to Seating → Guest List
    Depends on: Step 6
  8. Add Guest Record relation to RSVP Responses → Guest List
    Depends on: Step 1
  9. Add RSVP Response relation to Guest List → RSVP Responses
    Depends on: Step 8
  10. Create Music database
  11. Create Tasks database (with self-relation Depends On)
  12. Seed milestone tasks from task seeds
    Depends on: Step 11
  13. Add formulas to all databases
    Depends on: Steps 1–11
  14. Add rollups to Guest List + Seating
    Depends on: Steps 6–9, 13
  15. Create all views per database (35 total)
    Depends on: Steps 1–14
  16. Create row templates per database
    Depends on: Steps 1–14
  17. Create Day-of Timeline page from template
  18. Create Shot List page from template
  19. Restructure hub page with dashboard layout
    Depends on: Steps 1–18
  20. Add inline linked database views to hub page
    Depends on: Step 19

Day-of & Shot List

Two standalone Notion pages (not databases) shared directly with vendors.

A narrative document with time blocks from 6 AM through midnight. Shared with all vendors and the wedding party. Includes emergency contacts at the top.
